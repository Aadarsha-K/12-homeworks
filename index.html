Database Management System

A Database Management System (DBMS) is software designed to manage, retrieve, and manipulate data in a database. It enables users to create and maintain a database. Below are some terms commonly used in DBMS:

  • Data

    Data is raw information that can be processed by a computer. Data is the raw facts that, after processing, become information. Example: Kathmandu, 20, etc.

  • Information

    Information refers to data that has been processed and analyzed to provide meaningful output.

  • Metadata

    Metadata is the data that provides information about other data. It provides context and descriptions about data.

  • Data Dictionary

    A data dictionary is a centralized repository that provides detailed definitions and attributes for data elements used within a database.

  • Database

    A database is an organized collection of structured information stored electronically, typically managed by a Database Management System (DBMS). It serves as a repository of related information about a subject. Examples include businesses using databases to store customer information, schools managing student records, and online stores tracking product inventories.

  • Table

    A table in a database organizes data into rows and columns.

  • Tuple(s)

    A tuple in databases refers to a single row within a table that contains a specific set of data.

  • Field(s)

    Fields are individual pieces of data within a database table that represent attributes of records.

  • Domain

    The domain specifies the complete set of values that an attribute or column can contain within a database table.

  • Constraint

    Constraints enforce rules on data columns to ensure accuracy, integrity, and validity.

  • Key(s)

    Key is an attribute used to uniquely identify records in a table. Primary keys, foreign keys, and alternate keys are some types of keys used in databases.
